# item 10. 단위 테스트를 만들어라

### 단위 테스트가 확인하는 항목

- 일반적인 유스케이스 : 요소가 사용될 거라고 예상되는 일반적인 방법을 테스트함.
- 일반적인 오류 케이스와 잠재적인 문제 : 제대로 동작하지 않을거라고 생각하는 일반적인 부분, 과거에 문제가 발생했던 부분 테스트
- edge 케이스와 잘못된 아규먼트 : Int의 경우 Int.MAX_VALUE를 테스트하거나 nullable의 경우 null 테스트를 해보는 것을 의미함.

<br/>

### 단위 테스트의 장단점

- 장점
    - 테스트가 잘 된 요소는 신뢰할 수 있음.
    - 리팩터링에 두렵지 않음.
    - 수동으로 테스트하는 것보다 단위 테스트로 확인하는 것이 더 빠름
- 단점
    - 단위 테스트를 만드는 데 시간이 소요됨.
    - 테스트를 활용할 수 있게 코드를 조정해야 함.
    - 좋은 단위 테스트를 만드는 작업이 어려움.
